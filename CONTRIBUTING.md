# Contributing to CivicPulse

Thanks for your interest in contributing! We welcome fixes, features, documentation improvements, and tests.

Before you contribute, please read the guidelines below.

## How to contribute

1. Fork the repository and clone your fork.
2. Create a descriptive branch from `main`:
   - Feature: `feature/<short-description>`
   - Bugfix: `fix/<short-description>`
   - Docs: `docs/<short-description>`

3. Make commits with clear messages. We recommend using Conventional Commits style:
   - `feat:` a new feature
   - `fix:` a bug fix
   - `docs:` documentation only changes
   - `chore:` changes to build processes or auxiliary tools

4. Push your branch and open a pull request against `main` with a clear description.

## Code style and tests

- Keep code consistent with existing patterns (JavaScript, React, Express).
- Run linters/formatters if present. If none are configured, follow Prettier-style formatting.
- Add tests for new features where practical and verify existing behavior isn't broken.

## Pull Request Process

- Open a PR to `main` and include a short summary and the motivation.
- Link any related issue(s) or provide reproduction steps for bugs.
- Respond to review comments and update the PR as requested.

## Reporting security issues

If you've found a security vulnerability, please do not open a public issue. Contact the maintainers privately (add an email or security contact here).

## Code of Conduct

By participating, you agree to follow the project's Code of Conduct. Be respectful, constructive, and collaborative.

Thank you for contributing — we appreciate your help in making CivicPulse better!
